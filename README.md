## Key Features

### Autonomous AI Agents

Deploy intelligent trading agents that operate 24/7 without manual intervention. Each agent follows a configurable strategy, validates risk limits, and reports performance metrics in real time.

### Strategy Marketplace

Browse, deploy, and publish algorithmic trading strategies. Developers earn revenue when their strategies are used by other platform participants.

### AI Strategy Publishing

A dedicated SDK allows strategy developers to create, test, and publish strategies to the marketplace. Published strategies go through performance tracking and community ranking.

### Strategy Reputation & Ranking

Every strategy accumulates a reputation score based on real trading performance, risk-adjusted returns, and user adoption. Rankings help users discover high-quality strategies.

### Strategy Revenue Sharing

Strategy authors receive a share of fees generated by their published strategies. The revenue-sharing system is transparent and on-chain.

### Telegram Mini App Interface

The primary product interface is a Telegram Mini App — no separate app installation required. Users browse strategies, deploy agents, and monitor their portfolio directly inside Telegram.

### Core Components

| Component | Path | Description |
|---|---|---|
| **Agent Runtime** | `src/agent-runtime/` | 9-step execution pipeline: fetch data → load memory → call AI → validate risk → generate plan → simulate tx → execute on-chain → record outcome → update analytics |
| **Strategy Engine** | `src/strategy-engine/` | Trend (SMA), Arbitrage (spread detection), AI Signal (RSI/MACD) |
| **Market Data** | `src/market-data/` | CoinGecko and Binance providers with 30s TTL cache and automatic fallback |
| **Trading Engine** | `src/trading-engine/` | Simulated buy/sell execution, portfolio balance tracking, PnL calculation |
| **Portfolio Analytics** | `src/portfolio-analytics/` | Portfolio value, PnL, ROI, equity curve, trade history, risk monitoring |
| **Agent Control API** | `src/agent-control/` | REST API: list, start, stop, restart agents |
| **Strategy Marketplace** | `src/strategy-marketplace/` | Strategy discovery, publishing, ranking, revenue distribution |
| **Reputation Engine** | `src/reputation/` | Performance-based strategy scoring and ranking |
| **Revenue Sharing** | `src/revenue/` | On-chain revenue distribution for strategy authors |
| **Telegram Mini App** | `miniapp/` | HTML/CSS/JS dashboard — portfolio view, agent management, strategy selection |

For deeper technical details see [docs/architecture.md](docs/architecture.md).

## TON Wallet Integration

The platform implements a two-identity-layer authentication model:

```
telegram_user_id  +  wallet_address
       ↓                   ↓
Telegram identity    On-chain identity
```

### Connection Flow

```
User opens Mini App
        ↓
Telegram initData verified (HMAC-SHA256)
        ↓
User taps "Connect Wallet" in header
        ↓
Wallet selector opens (Tonkeeper / OpenMask / MyTonWallet / TON Space)
        ↓
User approves connection in wallet app
        ↓
Wallet address returned and validated
        ↓
Address persisted to user profile in DB (POST /api/wallet)
        ↓
Header shows shortened address (EQC…A1B2)
```

### Supported Wallets

| Wallet | Integration Method |
|---|---|
| **Tonkeeper** | JS Bridge (in-app browser) + Universal Link |
| **OpenMask** | JS Bridge |
| **MyTonWallet** | JS Bridge + Universal Link |
| **TON Space** | JS Bridge + Universal Link |

### Frontend Files

| File | Purpose |
|---|---|
| `telegram-miniapp/public/js/ton-connect.js` | TON Connect 2.0-compatible wallet module |
| `telegram-miniapp/public/index.html` | Wallet selector modal and connected-state UI |
| `telegram-miniapp/public/js/app.js` | Wallet state management and UI binding |
| `telegram-miniapp/public/css/miniapp.css` | Wallet UI styles |

### Backend API

**POST `/api/wallet`** — authenticated via `X-Telegram-Init-Data` header.

| Action | Description |
|---|---|
| `connect` | Link wallet address to user; stores `wallet_address` and `wallet_connected_at` |
| `disconnect` | Remove wallet link from user record |
| `status` | Return current wallet connection state for the user |

### Database Schema

The `taa_users` table stores wallet data:

```sql
wallet_address      VARCHAR(128)  -- TON address in user-friendly format
wallet_connected_at DATETIME      -- Timestamp of last successful connection
```

The `taa_agents` table tracks agent ownership:

```sql
owner_wallet  VARCHAR(128)  -- TON wallet of the agent owner (set at deploy time)
```

### Security

- Wallet connection requests are authenticated with Telegram `initData` verified via HMAC-SHA256.
- Wallet address format is validated server-side before storage (user-friendly or raw TON format).
- Session storage is used client-side to persist the connection within the Mini App session.
- The backend never stores private keys; only the public wallet address is persisted.

## Agent Runtime

Each agent follows a deterministic 9-step execution loop:

```
fetch_market_data
       ↓
load_agent_memory
       ↓
call_ai_model
       ↓
validate_risk
       ↓
generate_trade_plan
       ↓
simulate_transaction
       ↓
execute_trade
       ↓
record_results
       ↓
update_analytics
```

### Quick Start

```typescript
import { createMVPPlatform } from '@tonaiagent/core/mvp-platform';

// Initialize the platform
const platform = createMVPPlatform({ environment: 'simulation' });
platform.start();

// Create an AI agent
const agent = await platform.createAgent({
  userId: 'telegram_user_123',
  name: 'My Trend Agent',
  strategy: 'trend',       // 'trend' | 'arbitrage' | 'ai-signal'
  budgetTon: 1000,
  riskLevel: 'medium',     // 'low' | 'medium' | 'high'
});

// Start the agent
await platform.startAgent(agent.agentId);

// Execute a strategy cycle
const cycle = await platform.executeAgentCycle(agent.agentId);
console.log('Signal:', cycle.signal, '— trade executed:', cycle.tradeExecuted);

// Monitor portfolio
const metrics = await platform.getPortfolioMetrics(agent.agentId);
console.log('Portfolio value:', metrics.portfolioValue, 'TON');
console.log('PnL:', metrics.pnl, 'TON (', metrics.roi.toFixed(2), '% ROI)');

// Stop the agent
await platform.stopAgent(agent.agentId);
platform.stop();
```

See [docs/agent-runtime.md](docs/agent-runtime.md) for full agent runtime documentation.

## Strategy Marketplace

The strategy marketplace connects strategy developers with users:

```
Developer publishes strategy
         ↓
Strategy appears in marketplace
         ↓
Users browse and deploy strategy
         ↓
Performance is tracked automatically
         ↓
Developer earns revenue share
```

### Strategy Lifecycle

1. **Create** — develop a strategy using the Developer SDK
2. **Test** — backtest against historical data
3. **Publish** — submit to the marketplace
4. **Rank** — reputation score accumulates from live performance
5. **Monetize** — earn revenue when other users deploy the strategy

### Strategy Discovery

Users can filter strategies by:
- Risk level (low / medium / high)
- Historical return
- Reputation score
- Asset class (TON, BTC, ETH, ...)
- Strategy type (trend, arbitrage, AI signal)

## Portfolio & Monitoring

Every agent maintains a complete record of its trading activity and portfolio state.

### Tracked Metrics

- **Portfolio value** — current total value in TON
- **PnL** — absolute and percentage profit/loss
- **ROI** — return on investment since agent creation
- **Equity curve** — portfolio value over time
- **Trade history** — every executed trade with timestamp, asset, size, and outcome
- **Win rate** — percentage of profitable trades
- **Risk metrics** — max drawdown, volatility, Sharpe ratio

### Monitoring Dashboard

The Telegram Mini App displays all metrics in real time. Agents can also be monitored via the REST API:

```typescript
// GET /api/agents — list all agents with status
const agents = await platform.handleControlRequest('GET', '/api/agents');

// GET /api/agents/:id — get agent details and metrics
const status = await platform.handleControlRequest('GET', `/api/agents/${agentId}`);
```

See [docs/architecture.md](docs/architecture.md) for full monitoring documentation.

## Developer SDK

Developers can create and publish custom strategies using the platform SDK.

### Create a Strategy

```typescript
import { createStrategyEngine } from '@tonaiagent/core/strategy-engine';

const engine = createStrategyEngine();

// Register a custom strategy
engine.registerStrategy({
  id: 'my-custom-strategy',
  name: 'My Custom Strategy',
  description: 'A custom momentum-based strategy',
  riskLevel: 'medium',
  execute: async (marketData, portfolio) => {
    // Your strategy logic here
    const signal = computeSignal(marketData);
    return {
      action: signal > 0 ? 'buy' : signal < 0 ? 'sell' : 'hold',
      confidence: Math.abs(signal),
      amount: portfolio.balance * 0.1,
    };
  },
});
```

### Publish to Marketplace

```typescript
import { createMarketplace } from '@tonaiagent/core/strategy-marketplace';

const marketplace = createMarketplace();

await marketplace.publishStrategy({
  strategyId: 'my-custom-strategy',
  authorId: 'developer_wallet_address',
  revenueSharePercent: 20,
  minDeploymentBudget: 100,
});
```

## Deployment

### Prerequisites

```bash
node >= 18.0.0
npm >= 8.0.0
PHP >= 8.0        # for the Telegram Mini App backend
MySQL >= 8.0      # or MariaDB >= 10.3
```

### Local Development

```bash
git clone https://github.com/xlabtg/TONAIAgent.git
cd TONAIAgent
npm install
npm test          # verify all tests pass
```

### Standard PHP Hosting (Recommended for MVP)

The platform includes a step-by-step web installer for standard shared hosting:

1. Upload the `installer/` folder to your server
2. Visit `https://yourdomain.com/installer/` in a browser
3. Follow the setup wizard:
   - Configure database credentials
   - Enter your Telegram Bot Token (from [@BotFather](https://t.me/BotFather))
   - Set the Telegram webhook URL
   - Complete environment configuration
4. The installer configures everything automatically

### Requirements for Telegram Mini App

- HTTPS is required (Telegram Mini Apps enforce TLS)
- Apache with `mod_rewrite` or Nginx
- Telegram Bot Token from [@BotFather](https://t.me/BotFather)

### Telegram Mini App — Automated Deployment

The `telegram-miniapp/` package ships with two scripts that fully automate the setup:

```bash
cd telegram-miniapp

# 1. Fill in credentials
cp .env.example .env && nano .env

# 2. Deploy the static frontend (choose a mode)
./scripts/deploy-miniapp.sh vercel       # → Vercel
./scripts/deploy-miniapp.sh cloudflare   # → Cloudflare Pages
./scripts/deploy-miniapp.sh docker       # → Docker + Nginx
./scripts/deploy-miniapp.sh php          # → PHP server via SSH

# 3. Wire up the Telegram bot (webhook, menu button, commands)
./scripts/setup-bot.sh
```

See [`telegram-miniapp/README.md`](telegram-miniapp/README.md) for the complete deployment guide including deep links, security validation, and production hosting options.

### Vercel / Cloud Deployment

See [docs/deployment.md](docs/deployment.md) for cloud deployment instructions including Vercel, Docker, and VPS configurations.

## Long-Term Vision

TONAIAgent aims to evolve into an AI-native financial infrastructure where autonomous agents deploy capital, discover strategies, and coordinate algorithmic markets at scale.

The long-term protocol direction includes:

- **On-chain strategy registry** — immutable, verifiable strategy publishing on TON
- **Decentralized governance** — DAO-based platform governance with token voting
- **Cross-chain expansion** — bridging strategies and liquidity across chains
- **Institutional integrations** — custody, compliance, and reporting tools for institutional participants
- **AI coordination layer** — multi-agent systems that collectively optimize market efficiency

This vision is a multi-year roadmap. The current MVP and marketplace features form the foundation for this broader ecosystem.
