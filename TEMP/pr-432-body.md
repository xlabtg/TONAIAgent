## Логический ре-аудит TONAIAgent v2.43.0 (Issue #431)

Второй независимый проход по всей логике приложения (~976 файлов TypeScript + контракты Tact), запрошенный в #431 «Check via Claude Fable». Аудит **намеренно не повторяет** находки первого обзора: LOGIC-01..22 (issues #386–#407) проверены как **исправленные** в текущем дереве, поэтому нумерация продолжена с **LOGIC-23**.

Каждая находка подтверждена чтением фактического исходника по указанному пути и диапазону строк. Один кандидат (`services/regulatory/ai-governance.ts` «fail-open») исследован и **отброшен** как ложноположительный — пропуск отсутствующего под-условия оставляет надзор сработавшим (fail-closed), а не наоборот.

### Сквозная тема: «Built but not wired» (повторяется)

Главное наблюдение первого обзора сохраняется: средства безопасности реализованы и покрыты тестами, но путь, который должен их активировать, сломан.

- **LOGIC-25** — редактирование PII реализовано, но детектор отдаёт `warn` именно когда редактирование включено, а движок редактирует только при `block` → в дефолтной конфигурации PII не редактируется.
- **LOGIC-24** — пороговая подпись считает `verified` по каждой подписи, но в кворуме использует длину массива → неверифицированные подписи засчитываются.
- **LOGIC-23** — кворум одобрения человеком считает строки, а не уникальных одобряющих → один человек закрывает гейт N-из-M.
- **LOGIC-31 / 45 / 47** — подтверждение транзакции, выдача воркера и нарезка iceberg имеют «best-effort/TODO» сокращения (синтетический успех, переиспользование занятого воркера, завершение только по факту филла), молча обходящие гарантию.

### Содержимое PR

- `AUDIT_REPORT_TONAIAgent_v2.43.0_LOGIC_REVIEW_v2.md` — сопроводительный отчёт (executive summary, таблица severity, методология, индекс, план из 5 этапов).
- `TEMP/logic-review-v2/` — 29 самодостаточных документов находок (LOGIC-23..51) с problem/evidence/impact/suggested fix/acceptance criteria + README-индекс с маппингом LOGIC→issue.
- `experiments/logic-review-v2/` — генератор документов, скрипт подачи issue и **3 скрипта воспроизведения**, демонстрирующие дефекты в изоляции:
  - `repro-LOGIC-28-portfolio-overallocation.mjs` → переаллокация капитала (sum фракций = 1.5, выдача 1.5× баланса);
  - `repro-LOGIC-47-iceberg-infinite-loop.mjs` → бесконечный цикл на неисполненном лимитном ордере;
  - `repro-LOGIC-32-phantom-default-recovery.mjs` → фантомное полное покрытие дефолта при действующем потолке (скрыто 4.7M непокрытого убытка).

### Как воспроизвести

```bash
node experiments/logic-review-v2/repro-LOGIC-28-portfolio-overallocation.mjs
node experiments/logic-review-v2/repro-LOGIC-47-iceberg-infinite-loop.mjs
node experiments/logic-review-v2/repro-LOGIC-32-phantom-default-recovery.mjs
```

Каждый завершается с кодом 1 при воспроизведённом дефекте и печатает «BUG REPRODUCED».

### Примечание

Это автоматический логический аудит, а не профессиональный аудит безопасности человеком; перед запуском с реальными средствами требуется экспертная проверка. Несколько находок **High** поднимаются до *Critical* в продакшене с реальными средствами (например, LOGIC-28 переаллокация, LOGIC-29 выплаты казны, LOGIC-31 фантомные кросс-чейн ноги, LOGIC-24 неверифицированные подписи).
